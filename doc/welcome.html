Welcome to Stjörnhorn v0.1.44

A node-based image- and video-processing playground.

Getting started

  1. Create a flow — type a name and press Create, or open an existing .flowjs.
  2. Drop nodes from the list on the left onto the canvas.
  3. Wire them up by dragging output ports to input ports.
  4. Press Run and watch the viewer panel update.

Node categories

Sources

Load still images (JPEG, PNG, CR2) or video files (MP4, AVI, MOV, MKV) as streams of frames.

Filters

Scale, shift, blur, threshold, dither, split/join RGBA channels, overlay with per-pixel alpha, and match templates with NCC.

Sinks

Write frames to image files or encode video (MP4V, XVID) with live preview via the Display node.

What's new in v0.1.44

  • Checkbox check glyph restored. Same Qt stylesheet quirk again — the ::indicator:checked rule was setting fill colour but not painting an actual check mark, so a checked box just turned blue without showing a tick. A new checkmark.svg asset (white stroke, 14×14) is wired into the indicator, so the box now shows a proper check when toggled on.
  • Combobox dropdown chevron restored (v0.1.43) — same Qt stylesheet quirk that hid the spinbox arrows in 0.1.41 was hiding the picker arrow on every QComboBox.
  • Live-preview auto-run removed (v0.1.42) — editing a param (e.g. clicking a spinner) no longer kicks off a flow run. Re-runs are now strictly Run-button driven.
  • Spinbox up/down buttons show their arrow chevrons (v0.1.41) — two small SVG arrow assets ship in assets/icons/ now and the QSS injects their absolute paths at theme-apply time, plus hover / pressed states for tactile feedback.
  • Param widgets line up along a uniform X (v0.1.40) — within a node, every inline editor starts at the same horizontal anchor.
  • Param widgets are compact and consistent across OS styles (v0.1.39) — spinboxes, line edits and combo boxes lock at a Blender-style 22 px tall.
  • Internal cleanup (v0.1.38) — widget-sizing magic numbers consolidated into named constants.
  • Outputs at top, inputs at bottom (v0.1.37) — node sockets are no longer paired side-by-side row-for-row — output sockets stack at the top of the body (right-aligned), input sockets follow underneath (left-aligned, with their inline widgets). Visual layout matches Blender's geometry-nodes / shader editor.
  • Spinbox arrows no longer overlap the value field. The dark-theme stylesheet was setting padding on QSpinBox / QDoubleSpinBox without reserving room for the up/down buttons; Qt's style engine drew the buttons on top of the text. Padding now reserves a 16-px column on the right and the sub-controls are positioned explicitly to fill it.
  • Inline socket widgets render properly (v0.1.36) — tall enough rows that QSpinBox / QLineEdit show full-size up/down arrows and text caret, wide enough nodes that FilePathParamWidget's line-edit + Browse + View buttons stop overlapping each other.
  • Flow file format finalised (v0.1.35) — on-disk per-node params key renamed to port_defaults. Existing flow files still load — the loader reads the legacy key as a fallback.
  • Inline socket layout (v0.1.34) — param widgets sit directly on their input port's row, right of the port name. The separate property-panel block above the IO rows is gone — every editable property is a socket with an inline editor.
  • Param widgets gray out when their socket is driven (v0.1.33) — if you wire a Value Source into Overlay.angle, the slider for that param disables. Disabled state is captured at NodeItem creation; re-load the flow after a connect/disconnect to refresh.
  • The NodeParam class is gone. The UI now binds every editable widget to an InputPort directly: port.name drives the node attribute, port.metadata carries the widget hints, port.upstream reflects the connection state. Saved flows load identically; the old params property still exists on NodeBase but now returns the list of param-style input ports instead of NodeParam objects.
  • NodeParam declarations migrated to inline port-only form (v0.1.32) — every node in the catalog now declares its editable inputs as InputPort objects directly in __init__ with type / default / metadata; the per-node params property override is gone.
  • Overlay is a full param-as-port pilot (v0.1.31) — the manual "port if connected else param" branch in Overlay.process_impl is gone — the framework's auto-populate machinery covers it. Every numeric param (scale, angle, xpos, ypos, alpha) is drivable through its socket. Wire ramps in for sliding, zooming, fading or rotating animations.
  • Every NodeParam now has a matching input socket (v0.1.30) — auto-created from the param declaration with the right type (SCALAR for INT/FLOAT, BOOL/STRING/ENUM/PATH for the rest), so any numeric stream can drive any numeric param without each node having to declare its own port. Skip-mode follows Blender's mute semantics — short-circuit each output to the first type-compatible input — so the new param-style ports never accidentally make a node un-skippable.
  • Port-driven attribute population (v0.1.29) — before each process_impl, NodeBase writes any connected input port's current value into self._<port_name> and restores the user-set value afterwards.
  • Param-as-port groundwork (v0.1.28) — IoDataType now covers BOOL / STRING / ENUM / PATH alongside the existing image and numeric kinds, and InputPort grew a metadata dict for widget hints.
  • Sinks are no longer required (v0.1.27) — a flow that ends at a Display now runs without a downstream sink — the inline preview is the result. The throwaway Value Sink node (only there to satisfy the old "every flow needs a sink" rule) has been removed.
  • Overlay's angle is now port-drivable (v0.1.26) — Overlay grew an optional angle SCALAR input. Wire a Value Source (0..359) into it and the logo rotates a full turn per Run; leave it unconnected and the literal angle param works exactly as before. First pilot of the param-as-port mechanism — every numeric parameter is meant to grow a matching optional port over time.
  • Math nodes for scalars (v0.1.25) — new Math palette section in the filters list with a Math node (op selector: ADD, SUB, MUL, DIV, MIN, MAX) and a Clamp node (min_value, max_value). Plus a new Constant Value source that emits a single SCALAR (latched on streaming consumers) so you can feed a fixed factor or offset into Math without a second ValueSource.
  • Value Source (v0.1.24) — a numeric counter source that emits a SCALAR per frame (parameters: min_value, max_value, multiplier, loop). Wire it into a Display to watch the counter live in the inline preview.
  • Display now shows SCALAR & MATRIX (v0.1.24) — feed a scalar stream into Display and the inline preview renders the formatted number; matrices render as a compact text grid. Image payloads keep their pixmap preview, and the FPS read-out from v0.1.20 still fires for image streams.
  • SCALAR & MATRIX payload types (v0.1.23) — internal groundwork for value-driven nodes; existing flows are unaffected.
  • Node List is now a tree (v0.1.22) — each palette section is a collapsible group with expand-all / collapse-all buttons in the dock toolbar. The search box auto-expands matching groups so leaves never hide behind a closed section.
  • Directory Source (v0.1.21) — drop a folder of images into a flow as a frame stream, sorted lexicographically, with an include_subdirectories toggle for recursive walks.
  • Eight new processing nodes (v0.1.20) — Transform: Flip, Crop, Rotate. Processing: Gaussian Blur, Invert. Temporal: Frame Difference, Temporal Mean, Temporal Median.
  • FPS read-out on Display (v0.1.20) — the Display preview shows an exponentially-smoothed FPS counter in the top-left corner. Preview-only — anything written downstream by a sink stays clean.

Tips

  • Recent flows appear on the start page for quick access.
  • User-defined nodes live under ~/.image-inquest/user_nodes/.
  • Run with --flow NAME to jump straight into a saved flow.
  • Press F11 with the output inspector focused to toggle fullscreen preview.
